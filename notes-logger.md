# Notes Logger

## 目的
- チャットの往復をそのまま残し、仕様調査や検討の経緯を docs/notes に蓄積する。
- 命名と枠を揃えておき、次回以降はログ欄へ原文を追記するだけにする。

## 参照
- docs/notes/README.md（命名・棚卸し・サブディレクトリ運用）

## ファイル命名と設置
- 場所: `docs/notes/`
- 命名: `YYYYMMDD-<slug>.md`（例: `20251217-usage-consideration.md`）。slug は用途を英小文字ケバブで表す。`x回目` といった周回表現は付けない。
- RM 連動時は `rmxxx-<slug>.md` を採用。紐づけがなければ日付+用途のみ。

## 初回作成テンプレート
以下を貼って開始する。

```markdown
# <タイトル> (<日付>)
- 目的: <このメモで追う論点>
- スコープ: <想定範囲や対象ファイル>
- ログ方針: チャット原文をそのまま転記（整形・要約しない）
- メモ/タスク方針: 必要なら TODO を下部に置く

## ログ
- ユーザー: <原文>
- エージェント: <原文>

## TODO / メモ
- [ ] 追記があればここに箇条書き
```

## ログ追記手順
1. 対象ノートを開く（未作成なら上のテンプレートで新規作成）。
2. `## ログ` セクションにチャットの発言を時系列で追加する。整形や要約は行わず、原文をそのまま貼る。
3. 必要に応じて `## TODO / メモ` に未解決項目を箇条書きで追加する。
4. ノートを保存する。テストやコード変更は不要。

## 運用メモ
- チャット1回ごとにログを追記し、ファイル分割は日付＋用途単位で行う。
- 役割が増えたら発言者名だけ増やし、フォーマットは変えない。
- 長文化したら別ファイルに分け、元ノートに新ファイルへのリンクを記載する。
